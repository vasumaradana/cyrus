## Root Cause Analysis

The verification failures (`sh: 1: cyrus2/.venv/bin/python: not found`) were caused by a **barf-ts bug** (not a cyrus code issue):

- `autoCommand` in `packages/core/src/core/auto.ts` did not thread `projectCwd` through to `runLoop` or `verifyIssue`

- This caused relative paths in `.barfrc` (e.g., `cyrus2/.venv/bin/python`) to be resolved against barf's process CWD instead of the project directory

**Fixed by**: barf-ts issues 195 and 196 (centralized `projectCwd` threading), and cyrus issue 065 (systemic verification fix, state=COMPLETE).
