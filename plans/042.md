## Approach

Apply the pattern established by issue 040 systematically to all 35 remaining handlers. Each handler gets:
1. A `log.LEVEL("descriptive context", exc_info=True)` call
2. The appropriate level based on what the try block guards
3. Preserved control flow after the log call

**Why this approach**: Mechanical, consistent transformation. No behavioral changes — only adds observability. The `exc_info=True` pattern preserves full stack traces for debugging without changing error handling semantics.

## Rules to Follow

- No cyrus-specific `.claude/rules/` exist — follow general Python best practices
- `docs/16-logging-system.md` — Level mapping: ERROR for socket/IO/assertion failures, WARNING for timeouts/fallbacks, DEBUG for UIA walks/polling/cleanup
- `docs/12-code-audit.md` (H1) — Established fix strategy: add `logging.exception()` or `log.LEVEL(..., exc_info=True)` before pass/continue
- Pattern from issue 040 — `log.LEVEL("context", exc_info=True)` format, preserve control flow
- Always use `exc_info=True` — even on DEBUG, because when someone enables DEBUG they want full traces

## Key Decisions

- **exc_info=True on all levels**: Even DEBUG gets `exc_info=True` because these logs are only visible when DEBUG level is enabled, and when someone enables DEBUG they want full traces
- **Replace print() in handlers that already print errors**: Handlers at lines 264, 316, 899, 1453, 1482 already have `print(f"[error]...")` — these get replaced with `log.warning/error(...)` to avoid double-logging. Keep other print() calls that serve as status messages
- **Line 1040 is ERROR, not DEBUG**: Even though it's a window scan, it's the *initial* session scan at startup. Failure here means Cyrus starts with no sessions — that's a critical error, not a tolerated polling miss
- **Line 859 is WARNING, not DEBUG**: Clicking the "allow" button is an intentional user action. Failure means the permission dialog wasn't dismissed — the user needs to know
- **Issue 040 dependency**: 040 handles lines 200, 1033, 1082 but is still PLANNED. This plan adds `import logging` / `log = getLogger(...)` which 040 also needs. Whichever runs first adds the import; the second should detect it exists and skip
