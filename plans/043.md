## Key Decisions

- **Don't change log levels**: Issue 011 already categorized these handlers. The levels are correct per the categorization rules. No re-categorization needed.
- **Don't change logger name**: Issue 011 established `log = logging.getLogger("cyrus.voice")` with tests asserting it. The issue template suggests `__name__` but the project convention uses explicit names.
- **Keep `%s, e` args alongside `exc_info=True`**: The `%s` format gives a clean one-line message; `exc_info=True` adds the full traceback. Both are valuable — they serve different purposes.
- **No new tests needed**: Existing `test_011_cyrus_voice_logging.py` already validates the structural requirements. The fix only strengthens what's already tested.
