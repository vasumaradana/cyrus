## Priority Task: Fix 4 Worst Offenders

These are the highest-impact broad exception handlers identified in the code audit. Fixing these sets the pattern for remaining handlers.

## Target Handlers
1. **cyrus_brain.py:455** — UIA tree walk (DocumentControl check)
2. **cyrus_brain.py:462** — UIA tree walk (GetFirstChildControl loop)
3. **main.py:200** — Window tracking race condition in project switching loop
4. **main.py:1033–1034** — Session scan errors in polling loop
5. **main.py:1082** — VAD audio processing loop

## Approach
For each handler:
1. Analyze what exceptions could be raised in the try block
2. If specific exception types can be caught safely, use them (e.g., `except UIAutomationError:`)
3. If broad catch is necessary for resilience, add logging before pass/continue:
   - **UIA tree walks** (lines 455, 462) → `log.debug("UIA operation failed", exc_info=True)`
   - **Window tracking** (line 200) → `log.debug("Window tracking failed", exc_info=True)`
   - **Session scan** (line 1033–1034) → `log.error("Session scan failed", exc_info=True)`
   - **VAD loop** (line 1082) → `log.debug("VAD frame processing failed", exc_info=True)`

## Logging Setup
- Use standard Python logging: `import logging` and `log = logging.getLogger(__name__)`
- Logging infrastructure spec at `docs/16-logging-system.md` provides level mapping rules:
  - **ERROR**: Critical operations (session scan failures)
  - **DEBUG**: Tolerant/polling operations (UIA walks, window tracking, VAD)
- See `docs/16-logging-system.md` section "Print-to-log conversion rules" for exception handling patterns

## Acceptance Criteria
- [ ] All 4 worst offenders updated with appropriate logging
- [ ] UIA tree walks log at DEBUG level with `exc_info=True`
- [ ] Window tracking logs at DEBUG level with `exc_info=True`
- [ ] Session scan logs at ERROR level with `exc_info=True`
- [ ] VAD loop logs at DEBUG level with `exc_info=True`
- [ ] All exception handlers preserve original control flow (pass stays pass, continue stays continue)
- [ ] Code uses standard `logging` module (not custom infrastructure)
